# Judging Criteria

## Sheet

### Clinical AI Hackathon – Friday Judging Sheet (Clinicians Only)

Clinician judges should score only:

- Understanding (1-4)
- Solution (1-4)

This gives a clinician total of 2-8 per judge.

The result is by majority decision.

**Scoring Guide – use the same scale for both clinician categories**

1 = Not all core requirements implemented (major gaps; doesn't fully solve the task)

2 = All core requirements implemented (works end-to-end, accurate/useful Excel output, solid level)

3 = Surpassed expectations (noticeably better – clever features, polish, real-world ready)

4 = Blew my mind (exceptional – innovative, clinically transformative, wow factor)

**When to give a 4:** Use this sparingly for projects that genuinely surprise or impress you — e.g., unusually accurate/safe extraction on tricky real-world cases, smart clinician-friendly additions (like confidence flags or summaries), elegant handling of edge cases, or clear potential to save serious time/errors in NHS/MoD workflows. It should feel like "this could actually change how we work".
